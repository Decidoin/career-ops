# Mode: PDF

Triggered by: `/career-ops pdf` or after an evaluation when auto-generating a CV.

## Workflow

1. Read `cv.md` for master content
2. Read the evaluation report for the target role (or ask which role if ambiguous)
3. Read `templates/cv-template.html` for the HTML layout
4. Tailor the CV:
   - Reorder experience bullets to lead with most relevant for this role
   - Inject keywords from the JD naturally into achievement bullets
   - Customize the summary paragraph for this specific role and company
   - Remove or minimize experience that is irrelevant to this role
   - Apply the matching narrative template from `modes/_shared.md`
5. **Humanizer pass** — Before generating HTML, review the full tailored CV text and strip all AI tell-tale patterns:
   - Em dashes (—) used as connectors → replace with commas, colons, or restructure the sentence
   - Rule of three constructions ("X, Y, and Z" stacked repeatedly) → vary the rhythm
   - Filler openers: "Leveraging", "Spearheaded", "Passionate about", "Drove", "Championed", "Pioneered" → replace with plain active verbs (built, ran, improved, cut, increased, designed)
   - Passive voice inflation ("was responsible for", "was involved in") → active voice
   - Vague attributions ("collaborated with stakeholders to deliver insights") → specific verbs and outcomes
   - Inflated symbolism or generic claims with no numbers → either add a number or cut the claim
   - The final text must read as naturally written by Santosh, not generated by an AI. If a bullet sounds like an AI wrote it, rewrite it.
6. Generate HTML from the tailored, humanized CV content using `templates/cv-template.html`
7. Run `node generate-pdf.mjs` to convert HTML → PDF
8. Save to `output/{company-slug}-cv-{YYYY-MM-DD}.pdf`

## Tailoring Rules

- **Summary:** 2–3 sentences. Lead with the role-relevant angle (product analytics / BI / strategy / founder's office). Include the company name if possible.
- **Experience:** Reorder bullets within each role. Add JD keywords where they fit naturally.
- **Skills:** Mirror the JD's terminology. Put the most relevant tools first.
- **Publications:** Always include — LLM-powered BI paper is a differentiator.
- **Education:** Keep brief. Masters from Clark University and 3.9 GPA always included.
- **Length:** Strict 1 page.

## ATS Optimization

- No tables, columns, or fancy formatting in the underlying HTML
- Standard section headers: Summary, Experience, Publications, Education, Skills
- No images or icons
- Clean, semantic HTML that parses well
- Keywords from JD appear in context, not as a keyword dump
